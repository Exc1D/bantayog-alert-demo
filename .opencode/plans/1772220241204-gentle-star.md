# Bantayog Alert - UI/UX Improvements Plan

## Overview

This plan adds comprehensive UI/UX improvements to enhance usability, accessibility, and user engagement beyond responsive design.

---

## Priority 1: High-Impact Improvements

### 1. Dark Mode Support

**Location**: `src/App.jsx`, `src/App.css`, new theme context

**Changes**:

- Create `src/contexts/ThemeContext.jsx` for theme state management
- Add theme toggle button in Header component
- Extend Tailwind config with dark mode color palette
- Update all components to support dark/light variants using Tailwind `dark:` classes
- Persist preference in localStorage
- Default to system preference with manual override

**Components to update**:

- Header, Footer, TabNavigation
- FeedPost, FeedList, FeedFilters
- WeatherCard, WeatherGrid
- ProfileTab (AuthForm, UserProfile)
- ReportModal, ReportForm
- All modals and cards

---

### 2. Skeleton Loading States

**Location**: `src/components/Common/Skeleton.jsx` (new), existing components

**Changes**:

- Create reusable Skeleton component with variants (card, text, avatar, image)
- Replace LoadingSpinner usage in:
  - FeedList (skeleton cards while loading)
  - WeatherGrid (skeleton cards while loading)
  - ProfileTab (skeleton profile card)
- Add shimmer animation effect

---

### 3. Enhanced Empty States

**Location**: `src/components/Feed/FeedList.jsx`, `src/components/Weather/WeatherGrid.jsx`, `src/pages/FeedTab.jsx`, `src/pages/WeatherTab.jsx`

**Changes**:

- Create reusable EmptyState component in `src/components/Common/EmptyState.jsx`
- Add contextual illustrations/icons for each empty state
- Add suggested actions (e.g., "Be the first to report" for empty feed)
- Update FeedList empty state with helpful message
- Update WeatherGrid when no data available

---

### 4. Pull-to-Refresh

**Location**: `src/components/Common/PullToRefresh.jsx` (new), page components

**Changes**:

- Create PullToRefresh wrapper component using touch events
- Integrate into FeedTab, WeatherTab
- Add visual feedback (spinner/indicator)
- Works alongside existing auto-refresh logic

---

## Priority 2: Medium-Impact Improvements

### 5. In-App Notification Center

**Location**: `src/components/Common/NotificationCenter.jsx` (new), Header.jsx

**Changes**:

- Create NotificationBell component in Header
- Create NotificationCenter dropdown panel
- Store notification history in localStorage (or backend if available)
- Show recent alerts with read/unread status
- Badge count for unread notifications

---

### 6. Share Functionality

**Location**: `src/components/Feed/FeedPost.jsx`, `src/components/Common/ShareButton.jsx` (new)

**Changes**:

- Create ShareButton component using Web Share API
- Add share button to FeedPost cards
- Share report details as text/URL
- Fallback to copy link for unsupported browsers

---

### 7. Search Functionality

**Location**: `src/components/Feed/FeedFilters.jsx`, `src/components/Common/SearchInput.jsx` (new)

**Changes**:

- Add search input to FeedFilters
- Search by location (municipality, barangay), disaster type, description
- Debounced search input
- Clear search button
- Show "No results" state with suggestions

---

### 8. Improved Report Form UX

**Location**: `src/components/Reports/ReportForm.jsx`, `src/components/Reports/ReportModal.jsx`

**Changes**:

- Add progress stepper indicator showing current step
- Implement auto-save draft to localStorage
- Add photo preview with remove option before submit
- Add form validation feedback inline
- Optimize for large photo uploads (compress before upload)

---

## Priority 3: Accessibility & Polish

### 9. Accessibility Enhancements

**Location**: Multiple components

**Changes**:

- Add skip-to-main-content link in App.jsx
- Improve focus-visible states throughout
- Add more descriptive ARIA labels to interactive elements
- Improve color contrast for status badges (critical, moderate, minor)
- Add keyboard navigation support for map markers
- Add screen reader announcements for dynamic content

---

### 10. Confirmation Dialogs

**Location**: `src/components/Common/ConfirmDialog.jsx` (new), usage in components

**Changes**:

- Create reusable ConfirmDialog component
- Add confirmations for:
  - Report submission (before final submit)
  - Sign out
  - Profile deletion
  - Photo removal in report form

---

### 11. Weather Historical Data

**Location**: `src/components/Weather/WeatherCard.jsx`, `src/components/Weather/WeatherGrid.jsx`, `src/hooks/useWeather.js`

**Changes**:

- Extend weather hook to fetch 7-day forecast
- Add mini forecast strip to WeatherCard
- Create simple temperature trend visualization (optional)
- Show "feels like" temperature

---

### 12. Improved Map Controls

**Location**: `src/components/Map/LeafletMap.jsx`, `src/components/Map/MapControls.jsx`

**Changes**:

- Add map layer toggle (streets, satellite)
- Add map legend overlay showing marker types
- Add current filter status indicator on map
- Improve marker clustering for dense areas

---

## Implementation Order

1. **Week 1**: Dark Mode + Skeleton Loaders
2. **Week 2**: Empty States + Pull-to-Refresh + Accessibility basics
3. **Week 3**: Search + Share + Notification Center
4. **Week 4**: Report Form improvements + Confirm Dialogs
5. **Week 5**: Weather enhancements + Map improvements
6. **Week 6**: Testing, polish, bug fixes

---

## Testing Requirements

- Test dark mode across all screen sizes
- Verify skeleton states match actual content dimensions
- Test pull-to-refresh on actual mobile devices
- Test share functionality across browsers (Chrome, Safari, Firefox)
- Verify accessibility with screen readers (VoiceOver, NVDA)
- Test confirmation dialogs don't block user flow

---

## Notes

- All new components should follow existing code patterns and conventions
- Use Tailwind CSS for styling (consistent with existing codebase)
- Ensure new features work offline where applicable
- Maintain performance - avoid large bundle size increases
